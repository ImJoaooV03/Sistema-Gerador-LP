Seção 3 — Fluxos Principais

Dois fluxos core do sistema

Fluxo 1 — Upload e Extração de Design System

USUÁRIO
Faz upload de ZIP (index.html + assets)
API
Valida ZIP · extrai arquivos · salva no Storage (uploads/{id})
API
Cria job ds_extraction · status: pending · retorna job_id
IA ENGINE
Lê index.html · analisa classes, cores, tipografia, componentes
IA ENGINE
Gera design-system.html documentando: paleta · tipografia · espaçamento · componentes · animações
API
Salva design-system.html no Storage · atualiza job status: done
USUÁRIO
Vê preview do design-system.html gerado · opção de download · confirma e salva na biblioteca
API
Cria registro em design_systems + move arquivos para design-systems/{id}/

Fluxo 2 — Geração de Landing Page

USUÁRIO
Cria projeto: niche · sub_niche · page_type · briefing
API
Cria job lp_generation · status: pending
IA ENGINE
Filtra referências por niche + sub_niche + page_type · seleciona top 2–3 por similaridade
IA ENGINE
Lê index.html + design-system.html de cada referência selecionada
IA ENGINE
Extrai estrutura de seções · recombina dobras · adapta conteúdo ao briefing · gera HTML final
API
Salva html_output + sections JSON em paginas_geradas · atualiza job status: done
USUÁRIO
Vê preview (iframe + nova aba) · edita seções (código ou IA) · faz download do HTML

Editor de Seções — Pós-Geração

Modo Código
Editor HTML com syntax highlight
por seção individual
Salva e re-renderiza preview
Modo IA
Usuário descreve alteração
IA reescreve apenas aquela seção
Mantém classes do design system
